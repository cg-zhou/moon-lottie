# SVG 渲染器待办事项

本文档记录当前 `moon-lottie` SVG 渲染器在 demo 调试阶段的后续工作，方便对照 demo 页面里的 canvas / svg 开关逐项推进。

## 当前目标

- 让 demo 页面可以直接切换 MoonLottie 与 lottie-web 的渲染器，便于逐帧对照。
- 先把 SVG 渲染结果的可观察性补齐，再逐步提升 SVG 输出正确性。

## 待办清单

- [ ] 建立一份 SVG 渲染专项样例清单，优先覆盖蒙版、track matte、渐变、图片资源、文本、预合成和表达式。
- [ ] 梳理 MoonLottie SVG 与 lottie-web SVG 的已知差异，并把问题按“布局错误 / 样式错误 / 缺失元素 / 时间轴错误”分类。
- [ ] 为 SVG 渲染补充更稳定的逐帧回归验证，避免只靠手工在 demo 页面观察。
- [ ] 继续补齐 SVG renderer 对描边样式、虚线、混合模式、图层合成等未完成能力的支持说明与实现。
- [ ] 明确图片资源在 SVG 输出中的策略，确认外链资源、data URL 与导出场景的兼容性边界。
- [ ] 评估文本输出在字体、转义、对齐和字形一致性上的差异，必要时补充更稳妥的转义和回退策略。
- [ ] 评估表达式动画在 SVG 路径中的可用性，确认与当前 JS 表达式宿主的联动是否稳定。
- [ ] 视验证结果决定是否把 SVG 渲染链路抽成 browser-player 的正式能力，而不是只用于 demo 调试。
